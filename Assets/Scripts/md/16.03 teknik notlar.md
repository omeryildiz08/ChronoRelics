- `BaseManager`, level ödüllerini inventory'den base grid'e spawn ediyor.

- `AnomalyManager`, merge event'lerini dinleyip anomaly etkilerini uyguluyor.

## Teknik Mimari Özeti

### Grid ve Merge
- `GridManager`, grid verisini `GridTileData[,]` içinde tutuyor.
- Merge kontrolü 4 yönlü bağlı komşuluk üzerinden yapılıyor.
- Aynı item'dan bağlı en az 3 obje varsa `NextLevelItem` oluşturuluyor.
- Merge sonunda `OnMergeCompleted` event'i yayınlanıyor.

### Input
- `MergeableObject`, `OnMouseDown`, `OnMouseDrag`, `OnMouseUp` ile çalışıyor.
- Obje doğrudan merge yapmıyor; sadece `GridManager.TryMergeOrPlace(...)` çağırıyor.

### Save/Load
- Base sahnesindeyken tile lock durumu, obje yerleşimi ve inventory kaydediliyor.
- Level sahnesindeyken sadece inventory güncelleniyor.
- Save verisi `savegame.json` içine yazılıyor.

### Base Akışı
- Base açıldığında save yükleniyor.
- Inventory içindeki ödül item'ları ilk boş uygun grid'lere yerleştiriliyor.
- Yerleşen ödüller inventory listesinden kaldırılıyor.

### Level Akışı
- `LevelManager`, merge sonucu çıkan item hedef item ise level'ı bitiriyor.
- Ödül item inventory'ye ekleniyor.
- Win panel açılıyor ve oyuncu base sahnesine dönebiliyor.

### Quest Sistemi
- `LevelQuestManager`, merge event'ini dinliyor.
- Şu an sadece `ProduceItem` görev tipi ilerliyor.
- `QuestUI_Item`, görev kartı ve progress göstergesini yönetiyor.

### Anomaly Sistemi
- Type 1: merge sonrası rastgele bir açık tile kilitlenebiliyor.
- Type 2: önceden belirlenmiş anomaly orb tile'ları üzerinden etkiler çalışıyor.
- Type 2 etkileri:
  - Açık tile kilitleme
  - Kilitli tile içindeki objeyi açık tile'a taşıma
  - Farklı era item spawn etme
  - Orb teleport etme

## İleride Dönüp Bakılacak Teknik Riskler

### 1. `Vector2Int.zero` sentinel kullanımı
- `MergeableObject.InitializeObject()` içinde `CurrentGridPosition == Vector2Int.zero` kontrolü var.
- Eğer `(0,0)` grid üzerinde gerçekten kullanılan geçerli bir tile ise bu yaklaşım ileride hataya açık.

### 2. Merge sonrası initialization akışı kırılgan
- `GridManager.PerformMerge()` yeni objeyi register ediyor ama açık biçimde `InitializeObject()` çağırmıyor.
- Şu an davranış `Start()` üzerinden toparlanıyor gibi duruyor.
- Bu akış ileride prefab davranışına veya script execution order'a bağlı bug çıkarabilir.

### 3. Save sistemi sahne adına sıkı bağlı
- `SaveManager`, base kontrolünü doğrudan `"BaseScene"` string'i ile yapıyor.
- Sahne adı değişirse save davranışı kırılır.
- İleride sahne bazlı daha esnek bir yaklaşım gerekebilir.

### 4. Quest sistemi eksik kapsamda
- `MergeCountLimit` tanımlı ama çalışmıyor.
- Level tamamlanma koşulu ile quest tamamlama koşulları ileride çakışabilir.
- GDD'deki "ödülü almak için quest tamamlama" mantığı henüz kodda tam karşılık bulmuş değil.

### 5. Anomaly foreign item yaşam döngüsü belirsiz
- Anomaly ile spawn edilen item'lar `IsInactiveAnomalyItem = true` alıyor.
- Bu item'ların ne zaman temizleneceği, aktif hale gelip gelmeyeceği veya save/load'da nasıl davranacağı net değil.

### 6. Base ödül yerleştirme akışı genişletilmeye açık
- `BaseManager`, inventory'deki item'ları ilk boş uygun slotlara sırayla koyuyor.
- İleride ödül paneli, manuel yerleştirme, stack sistemi veya kapasite yönetimi istenirse bu akış değişecek.
